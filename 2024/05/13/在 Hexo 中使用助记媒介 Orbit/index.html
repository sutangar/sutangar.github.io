在 Hexo 中使用助记媒介 Orbit

简而言之

Orbit 是一种在线性文章中嵌入间隔重复记忆系统的助记媒介，使读者能够更轻松地理解和记住复杂的主题，本文介绍了它的用法。

Orbit 是在线性叙述的文章（比如博文）中嵌入间隔重复记忆系统的一种助记媒介。简单来说，助记媒介就是把常有的先有材料再制成抽认卡的过程的对象转移了：由面向读者变为了面向作者。

如果使用得当，这些助记卡片可以让读者可靠地记住阅读的内容，更轻松地理解复杂的主题，并随着时间的推移成为长期记忆的一部分。本文是基于 Hexo 的静态博客框架来使用 Orbit 的。

开始使用 Orbit

Orbit 被设计用于在网络上的任何位置工作。可以使用可在任何 HTML 页面中使用的 Web 组件嵌入 Orbit 。设置很简单：添加一个 <script>标签，就能在页面上的任何位置使用特殊标签<orbit-reviewarea>和 标签来插入助记卡片。

但这里我不使用文档提供的方法插入 Hexo 博文，而是直接在.md文件中直接写入 html 片段以降低发生意外的风险，这也使其更利于维护。在 Hexo 中使用 Orbit 非常简单，在博客博文源文件.md中写入上文的 html 片段即可实现。但请注意使用<html></html>包裹以避免出错。下面的代码片段例子分别演示了普通的卡片、带图片的卡片和使用 LaTeX 跨行与行内公式的卡片：

卡片颜色

复习卡片有一个color属性。每个站点都可以选择其 Orbit 助记卡片的颜色。这有什么作用呢？当读者查看来自多个站点的助记卡片时，他们会看到以各个站点选择的颜色呈现的助记卡片，从而提供一些上下文。我推荐使用不同于的颜色。支持的颜色值有： red、 orange、 brown、 yellow、 lime、 green、 turquoise、 cyan、 blue、 violet、 purple和 pink。替换颜色也可以使撰写者在单个页面上包含多个复习卡片集合。

结语

为什么要使用助记媒介？我个人觉得，原因不外乎两点：① 抽认卡的制作是一件并不轻松的工作（不但难以坚持、更难以产出高质量的卡片），而内容创作者自己来创作抽认卡则在一定程度上改善了这一状况；② 动态的互动性让博文更有意思了，也有助于间隔重复策略的推广。什么？你问我的想法？我个人当然是觉得用这种新鲜小玩意很好玩啦。

另外，你可以在量子国度直接体验助记媒介，快去试试吧。
